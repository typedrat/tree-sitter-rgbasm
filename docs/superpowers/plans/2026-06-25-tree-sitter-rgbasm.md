# tree-sitter-rgbasm Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a tree-sitter grammar for the RGBASM assembly language (RGBDS v1.0.1) plus highlight/structure queries for both nvim/Helix and Zed.

**Architecture:** A hand-written `grammar.js` parses RGBASM surface syntax (no preprocessor expansion). Instructions are generic (mnemonic + comma-separated operands); registers, condition codes, mnemonics, and directives are case-insensitive keyword tokens that win over identifiers via token precedence. The line-based structure is handled with an explicit newline token plus `extras` for whitespace, block comments, and line continuations — no external scanner in Phase 1. Two query sets (`queries/` for nvim/Helix, `queries/zed/` for Zed) are driven off the same node types.

**Tech Stack:** JavaScript (tree-sitter grammar DSL), `tree-sitter-cli` 0.26.9, C compiler (gcc, for generated `parser.c`), Node 24.

## Global Constraints

- **tree-sitter-cli version:** `0.26.9`, pinned as a `devDependency`. Never install globally; always invoke via `npx tree-sitter …`.
- **Do NOT run `npx tree-sitter init`** — it requires a TTY and fails in non-interactive shells. All metadata files are hand-written in Task 1.
- **Grammar name:** `rgbasm`. **Scope:** `source.rgbasm`. **File types:** `asm`, `inc`.
- **Reference grammar:** parses *surface syntax as written*; it does NOT model macro/`EQUS` text expansion or symbol interpolation.
- **Case rule:** reserved keywords (mnemonics, directives, registers, conditions, section types) are case-insensitive; identifiers are case-sensitive.
- **TDD:** every grammar change is driven by a `test/corpus/*.txt` entry that fails first. Corpus source sections must NOT start with a blank line (a leading blank line is parsed as source content).
- **Commit** after every green task step group, with the exact message given.
- **License:** MIT, author `Alexis Williams <alexis@typedr.at>`.
- **Phase-1 scope only.** Deferred (do NOT implement): symbol interpolation `{fmt:sym}`, raw strings `#"…"`, multi-line strings `"""…"""`, fragment literals `[[ … ]]`, anonymous labels `:+`/`:-`, graphics constants `` `0123 ``, charmap directives, full macro-arg interpolation semantics.
- **Spec:** `docs/superpowers/specs/2026-06-25-tree-sitter-rgbasm-design.md`.

```js
/**
 * RGBASM (RGBDS v1.0.1) grammar for tree-sitter.
 * Parses surface syntax; does not model macro/EQUS expansion or interpolation.
 */

// Case-insensitive regex for a reserved keyword, e.g. ci('LD') matches ld/LD/Ld.
function ci(word) {
  return new RegExp(
    word
      .split('')
      .map((c) => (/[a-z]/i.test(c) ? `[${c.toLowerCase()}${c.toUpperCase()}]` : c.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')))
      .join(''),
  );
}

// A single token matching any of the given case-insensitive keywords, with
// precedence 1 so it wins over `identifier` (precedence 0) on exact matches.
function kw(...words) {
  return token(prec(1, choice(...words.map(ci))));
}

// Comma-separated, non-empty list.
function sepByComma(rule) {
  return seq(rule, repeat(seq(',', rule)));
}

const PREC = {
  or: 1,
  and: 2,
  cmp: 3,
  add: 4,
  bitwise: 5,
  shift: 6,
  mul: 7,
  unary: 8,
  exp: 9,
  call: 10,
};

module.exports = grammar({
  name: 'rgbasm',

  extras: ($) => [
    /[ \t]/, // horizontal whitespace
    /\\\r?\n/, // line continuation: backslash + newline
    $.block_comment,
  ],

  rules: {
    // A file is newline-separated statements, tolerating blank lines anywhere
    // and an optional missing final newline.
    source_file: ($) =>
      optional(
        seq(
          repeat('\n'),
          $.statement,
          repeat(seq(repeat1('\n'), $.statement)),
          repeat('\n'),
        ),
      ),

    // Placeholder statement: replaced/extended in later tasks. For now a
    // statement is just a line comment (block comments are extras).
    statement: ($) => $.comment,

    comment: ($) => token(seq(';', /[^\n]*/)),

    // Non-nesting block comment; usable mid-line, so it lives in `extras`.
    block_comment: ($) => token(seq('/*', /[^*]*\*+([^/*][^*]*\*+)*/, '/')),
  },
});
```

- [ ] **Step 3: Implement the rules** — replace the `statement` placeholder and add rules

Replace the `statement` rule and add the following to the `rules` object:

```js
    statement: ($) =>
      choice(
        seq($.label_definition, optional($._line_body)),
        $._line_body,
        $.comment,
      ),

    // Placeholder body; alternatives added in later tasks.
    _line_body: ($) => $.comment,

    label_definition: ($) =>
      choice(
        // Global/scoped labels require a colon (distinguishes from macro calls).
        seq(field('name', $.identifier), field('export', optional('::')), optional(':')),
        // Local labels may omit the colon.
        seq(field('name', $.local_label), optional(choice('::', ':'))),
      ),

    identifier: ($) => token(/[A-Za-z_][A-Za-z0-9_#$@]*(\.[A-Za-z_][A-Za-z0-9_#$@]*)?/),

    local_label: ($) => token(/\.[A-Za-z_][A-Za-z0-9_#$@]*/),
```

Note on the global label `::` vs `:`: the form `seq(identifier, optional('::'), optional(':'))` lets `Name`, `Name:`, and `Name::` parse, but `Name` with no colon would collide with macro invocation (Task 7). To keep Task 2 unambiguous, require at least one colon for the identifier form by using `field('export', optional('::'))` followed by `optional(':')` **and** a guard: change the first choice arm to require a colon:

```js
        seq(field('name', $.identifier), choice('::', ':')),
```

Use this single-colon-or-double-colon arm (drop the separate `optional(':')`).

- [ ] **Step 5: Implement literals, expressions, and the `data_directive` host**

Add to the `rules` object, and add `$.data_directive` as an alternative of `_line_body` (change `_line_body` to `choice($.comment, $.data_directive)`):

```js
    data_directive: ($) =>
      seq(field('keyword', kw('DB', 'DW', 'DL', 'DS')), optional($.argument_list)),

    argument_list: ($) => sepByComma($.expression),

    expression: ($) =>
      choice(
        $.number,
        $.string,
        $.identifier,
        $.local_label,
        $.program_counter,
        $.parenthesized_expression,
        $.unary_expression,
        $.binary_expression,
        $.exp_expression,
        $.call_expression,
      ),

    program_counter: ($) => token('@'),

    parenthesized_expression: ($) => seq('(', $.expression, ')'),

    call_expression: ($) =>
      prec(
        PREC.call,
        seq(field('function', $.identifier), '(', optional($.argument_list), ')'),
      ),

    unary_expression: ($) =>
      prec(PREC.unary, seq(field('operator', choice('+', '-', '~', '!')), $.expression)),

    exp_expression: ($) =>
      prec.right(PREC.exp, seq($.expression, '**', $.expression)),

    binary_expression: ($) => {
      const table = [
        [PREC.mul, choice('*', '/', '%')],
        [PREC.shift, choice('<<', '>>>', '>>')],
        [PREC.bitwise, choice('&', '|', '^')],
        [PREC.add, choice('++', '+', '-')],
        [PREC.cmp, choice('===', '!==', '==', '!=', '<=', '>=', '<', '>')],
        [PREC.and, '&&'],
        [PREC.or, '||'],
      ];
      return choice(
        ...table.map(([p, op]) =>
          prec.left(
            p,
            seq(field('left', $.expression), field('operator', op), field('right', $.expression)),
          ),
        ),
      );
    },

    number: ($) =>
      choice($.hex, $.octal, $.binary, $.fixed_point, $.decimal, $.char_constant),

    // Order matters only for documentation; the lexer uses longest-match.
    decimal: ($) => token(/[0-9][0-9_]*/),
    hex: ($) => token(/(\$|0[xX])[0-9A-Fa-f][0-9A-Fa-f_]*/),
    octal: ($) => token(/(&|0[oO])[0-7][0-7_]*/),
    binary: ($) => token(/(%|0[bB])[01][01_]*/),
    fixed_point: ($) => token(/[0-9][0-9_]*\.[0-9][0-9_]*([qQ][0-9]+)?/),
    char_constant: ($) => token(/'(\\['"{}\\nrt0]|[^'\\\n])'/),

    string: ($) =>
      seq('"', repeat(choice($.escape_sequence, $._string_content)), token.immediate('"')),
    _string_content: ($) => token.immediate(prec(1, /[^"\\\n]+/)),
    escape_sequence: ($) => token.immediate(/\\['"{}\\nrt0]/),
```
